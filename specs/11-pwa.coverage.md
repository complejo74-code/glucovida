## Edge Cases
- **Offline → warm state, not blank/error** — **Done** — SW `navigate` handler
  falls back to cached `/offline` (`src/app/offline/page.tsx`) or, if that could
  not be cached, a self-contained inline branded HTML page (styles inline, no
  external CSS dependency). Warm rioplatense copy: "Sin conexión, ya volvemos".
- **Manifest must not break login/auth; session preserved** — **Done** —
  `start_url: /chat` relies on the existing `/`→session redirect; SW never
  caches navigations or `/api/*`, so auth cookies/session flow is untouched.
  Middleware left unmodified. `/offline` precache uses `redirect: "manual"` and
  refuses to cache redirected/non-OK responses, so a logged-in non-onboarded
  user's onboarding redirect can't poison the offline cache or abort install.

## Notes
- **No external image tooling available** (no sharp/ImageMagick in the repo), so
  icons are generated by a small self-contained PNG encoder script,
  `scripts/gen-pwa-icons.mjs` (committed). Re-running `node scripts/gen-pwa-icons.mjs`
  reproduces the four PNGs byte-for-byte.
- **Middleware deliberately untouched** (guardrail: don't touch auth/security).
  `/sw.js` and `/manifest.webmanifest` do pass through middleware but are public
  and serve correctly (verified 200 + correct content-types).
- **SW registered production-only** to avoid cache-first interfering with dev HMR.
- Pre-existing build warnings (multiple lockfiles, tailwind config module type)
  are unrelated to this step and were left as-is.
